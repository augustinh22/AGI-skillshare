GIT-Introduction

maptime salzburg (2018-12-12)

Steffen Reichel

What this talk is not

a comprehensive tour through all the details and intricacies of GIT

a replacement for the documentation

What this talk is

an overview of the most important ideas behind Git.

an explanation what the HUB in GitHub adds

a basic example on how to contribute to projects

More help and documentation

GitHub Guides (https://guides.github.com/) are a fantastic resource with a lot of examples, guides, and further links.

GitHub Help (https://help.github.com/) has all the GitHub details.

All the details about Git can be found at https://git-scm.com/doc

Forking a repository

If you want to contribute to a project the most important thing is to fork the repository you want to contribute to.

Clone a repository

To actually edit files you have to create a local copy (clone) of the repository. A good practise is to also create a branch for every feature you add.
